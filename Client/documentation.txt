১. Delete Handle তৈরি করা
ফোল্ডার: helpers/actions
একটি গ্লোবাল Delete Handle ফাংশন তৈরি করা হয়েছে, যা বিভিন্ন কম্পোনেন্টে ব্যবহার করা যাবে। এর বিস্তারিত কার্যপ্রণালী:

Delete Button Component:
DeleteBtn নামের একটি প্রি-ডিজাইন রিঅ্যাক্ট আইকন কম্পোনেন্ট তৈরি এবং ব্যবহার করা হয়েছে।

Function Details:

Dynamic Routing: ফাংশনটি ডিলিট করার জন্য শুধুমাত্র ডায়নামিক রাউট (route) গ্রহণ করবে।
API Hit: প্রাপ্ত রাউটের উপর ভিত্তি করে সার্ভারে API কল করবে।
Loading State:
ডিলিট প্রক্রিয়া চলাকালীন Spinner দেখাবে।
প্রক্রিয়া শেষ হলে Delete Button পুনরায় দেখাবে।
২. Edit Handle তৈরি করা
কার্যপ্রণালী:
Edit Handle এর জন্য একটি ফাংশন তৈরি করা হয়েছে, যা ডাইনামিক ডেটা সম্পাদনা করতে ব্যবহৃত হবে। এর কাজের ধাপ:

Editable Data (Data):
ফাংশনটি ডাইনামিক ভাবে সম্পাদনার জন্য প্রয়োজনীয় ডেটা গ্রহণ করবে।

Dynamic Path:
যে ফর্মে ডেটা যোগ বা সম্পাদনা করা হবে (Add/Edit Form), তার রাউট পাথ ডাইনামিক ভাবে ফাংশনে পাস করা হবে।

Implementation:
এই ফাংশনটি ফর্ম বা এডিটেবল UI-তে ব্যবহার করে সহজেই ডেটা সম্পাদনা বা আপডেট করা যাবে।

উপসংহার:
Delete এবং Edit হ্যান্ডলার গ্লোবাল ভাবে তৈরি করা হওয়ায়, কোড পুনঃব্যবহারযোগ্য এবং মডিউলার। এর ফলে ভবিষ্যতে নতুন ফিচার বা পরিবর্তন যোগ করা আরও সহজ হবে।
